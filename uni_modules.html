Year 1

Comp 105 - Programming Language Paradigms

This module was mostly learning a new programming style, as well as a new programming language. The name of the language is 'Haskell' and its paradigm is purely functional programming. Looks and feels completely different compared to imperative programming languages.
It did help me become a better programmer, and has widened my view of the different kinds of programming languages out there. But it is not a paradigm I continued using after the module finished.

Comp 108 - Data Sctructures and Algorithms

The name of the module is fairly self-explanatory. It explored arrays, queues, stacks, linked lists and different sorting algorithms comparing their efficiency. The module was mostly theory driven but these are very useful notions in computer science, some of which I applied in other modules in my course, along with a multitude of coding tests and challenges.

Comp 116 - Analytic Techniques for Computer Science

Many of the techniques learned in this module are used in fields such as Machine Learning, Data Science, Algorithm Analysis, Experimental Computing and Optimisation. A theory heavy module with complicated new concepts, some of which took time to fully grasp. It did widen my understanding of the related fields such as Data Science, but I haven't put this knowledge into practice as much as I would like to.

Comp 122 - Object Oriented Programming

This was one of my favorite modules, as it had a lot of Java coding in it, but also explored concepts which I found to be useful. Hierarchy of Related Classes, Polymorphism and GUI, all of which were also put in practice in this module. The knowledge and skill gained from COMP 122 led me to other interesting projects and making future modules easier.

Comp 124 - Computer Systems

An introduction to the hardware and software components of a modern computer system. Learning about the Linux Kernel, processes down to the CPU Register level, and coding in Assembly Language. Unfortunateley, I have yet to apply those outside of the module (other than Linux kernel), but regardless, it helped me understand more about the low-level innerworkings of modern systems.
